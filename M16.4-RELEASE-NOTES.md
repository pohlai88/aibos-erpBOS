# M16.4 Release Notes - Advanced Asset Management

## 🚀 **Version 16.4.0 - Enterprise Asset Management**

**Release Date:** November 2025  
**Tag:** `v16.4.0`  
**Type:** Major Feature Release

## ✨ **What's New**

### **M16: CAPEX & Depreciation**
- ✅ **Asset class management** with configurable depreciation methods
- ✅ **Straight-line (SL) and Double Declining Balance (DDB)** depreciation
- ✅ **CAPEX plan creation** with flexible asset configurations
- ✅ **Depreciation schedule generation** with precision control
- ✅ **Automated journal posting** with balanced entries

### **M16.1: Intangibles & Amortization**
- ✅ **Intangible asset management** (software, patents, trademarks)
- ✅ **Amortization schedule generation** with straight-line method
- ✅ **Flexible asset classes** for different intangible types
- ✅ **Automated amortization posting** with journal integration

### **M16.2: CSV Imports + Bulk Posting**
- ✅ **CSV import functionality** for both CAPEX and Intangibles
- ✅ **Header mapping** with flexible column configuration
- ✅ **Default value injection** for missing fields
- ✅ **Bulk posting** with dry-run capabilities
- ✅ **Idempotent operations** with source hash validation

### **M16.3: Advanced Operations**
- ✅ **Unpost/Repost functionality** with safety checks
- ✅ **Proration support** (days-in-month, half-month)
- ✅ **Asset impairment tracking** with journal integration
- ✅ **Configuration management** for policies and preferences
- ✅ **UI draft caching** for state management

### **M16.4: FX Presentation Lock**
- ✅ **FX rate snapshotting** for audit compliance
- ✅ **Presentation currency locking** (in-service vs post-month)
- ✅ **Audit trail** for all FX rate decisions
- ✅ **Multi-currency support** with rate history

## 🔧 **Technical Features**

### **API Endpoints**
- `GET/PUT /api/assets/config` - Assets configuration management
- `POST/GET /api/assets/impairments` - Impairment operations
- `POST /api/assets/unpost` - Safe unposting with validation
- `POST/GET/DELETE /api/assets/drafts` - UI draft management
- `POST /api/assets/posting/bulk/dry-run` - Enhanced dry-run
- `POST /api/assets/posting/bulk/commit` - Draft-based posting
- `POST /api/capex/plan/import` - CAPEX CSV import
- `POST /api/intangibles/plan/import` - Intangibles CSV import

### **Database Schema**
- **6 new tables** with proper constraints and indexes
- **Audit trail** for all asset operations
- **Idempotent design** with source hash validation
- **Performance optimized** with strategic indexing

### **Safety Features**

- ✅ **Dry-run enforcement** (blocks unsafe operations)
- ✅ **Duplicate detection** (prevents double-posting)
- ✅ **Audit logging** (complete operation history)
- ✅ **Rollback capability** (safe unposting)


## 🚀 **Getting Started**

### **Quick Start (10 minutes)**
1. **Run migrations** in order (0031-0043)
2. **Seed asset classes** and posting maps
3. **Configure RBAC** with `capex:manage` capability
4. **Test CSV import** with sample files
5. **Generate schedules** and test bulk posting
6. **Verify reports** show depreciation/amortization

### **Sample CSV Files**
- `capex_sample.csv` - CAPEX import template
- `intangibles_sample.csv` - Intangibles import template

### **API Testing**
```bash
# Test configuration
curl -X GET -H "X-API-Key: <key>" http://<host>/api/assets/config

# Test CSV import
curl -X POST -H "X-API-Key: <key>" \
  -F "file=@capex_sample.csv" \
  -F 'json={"defaults":{"currency":"MYR","present_ccy":"MYR","method":"SL"}}' \
  http://<host>/api/capex/plan/import

# Test bulk posting
curl -X POST -H "X-API-Key: <key>" -H "content-type: application/json" \
  -d '{"kind":"depr","year":2025,"month":11,"dry_run":true}' \
  http://<host>/api/assets/posting/bulk/dry-run
```
