**Zero churn to your M24 stack; this snaps onto your AR + cash-app engine and feeds M22 receipts in real time.**

## 🚀 What You Get

- **Self-serve portal** (magic-link) for customers to view/pay invoices
- **Card/ACH gateway adapters** (tokenized, PCI-safe), 3-D Secure ready
- **Partial payments, multi-invoice checkout, surcharges** (configurable)
- **Self-serve PTP & disputes** that flow into M24 workstreams
- **Webhooks → cash application** (auto-match) + **M22 receipt signals**
- **Receipts, e-mail dispatch, refunds/voids** via ops endpoints

## 🎯 API Endpoints

### Public Portal Routes (Token-based)

| Endpoint                       | Method | Purpose                                     |
| ------------------------------ | ------ | ------------------------------------------- |
| `/api/portal/init`             | POST   | Initialize portal session & send magic link |
| `/api/portal/invoices`         | POST   | Get customer invoices                       |
| `/api/portal/checkout/intent`  | POST   | Create checkout intent                      |
| `/api/portal/checkout/confirm` | POST   | Confirm checkout & process payment          |
| `/api/portal/ptp`              | POST   | Create promise-to-pay                       |
| `/api/portal/disputes`         | POST   | Create dispute                              |

### Webhook Routes (HMAC-verified)

| Endpoint                      | Method | Purpose                |
| ----------------------------- | ------ | ---------------------- |
| `/api/portal/webhooks/stripe` | POST   | Stripe webhook handler |
| `/api/portal/webhooks/adyen`  | POST   | Adyen webhook handler  |
| `/api/portal/webhooks/paypal` | POST   | PayPal webhook handler |

### Ops/Admin Routes (RBAC)

| Endpoint                   | Method   | Capability         | Purpose                 |
| -------------------------- | -------- | ------------------ | ----------------------- |
| `/api/ar/surcharge/policy` | GET/PUT  | `ar:portal:policy` | Manage surcharge policy |
| `/api/ar/portal/sessions`  | GET/POST | `ar:portal:ops`    | Manage portal sessions  |
| `/api/ar/checkout/refund`  | POST     | `ar:portal:ops`    | Process refunds         |
| `/api/ar/webhook/retry`    | POST     | `ar:portal:ops`    | Retry failed webhooks   |
| `/api/ar/checkout/intents` | GET      | `ar:portal:ops`    | List checkout intents   |

## 🔄 Workflow Examples

### Customer Payment Flow

1. **Initiate Portal Session**

   ```bash
   curl -X POST https://portal.yourdomain.com/api/portal/init \
     -H "Content-Type: application/json" \
     -d '{
       "customer_id": "CUST-001",
       "email": "customer@example.com",
       "ttl_minutes": 60
     }'
   ```

2. **Customer Views Invoices** (via magic link)

   ```bash
   curl -X POST https://portal.yourdomain.com/api/portal/invoices \
     -H "Content-Type: application/json" \
     -d '{
       "token": "abc123...",
       "include_paid": false
     }'
   ```

3. **Create Checkout Intent**

   ```bash
   curl -X POST https://portal.yourdomain.com/api/portal/checkout/intent \
     -H "Content-Type: application/json" \
     -d '{
       "token": "abc123...",
       "invoices": [{"invoice_id": "INV-001", "amount": 1000}],
       "present_ccy": "USD",
       "gateway": "STRIPE",
       "save_method": true
     }'
   ```

4. **Confirm Payment**
   ```bash
   curl -X POST https://portal.yourdomain.com/api/portal/checkout/confirm \
     -H "Content-Type: application/json" \
     -d '{
       "intent_id": "intent_123",
       "gateway_payload": {"payment_method": "pm_123"}
     }'
   ```

### Self-Serve PTP/Dispute Flow

1. **Create PTP**

   ```bash
   curl -X POST https://portal.yourdomain.com/api/portal/ptp \
     -H "Content-Type: application/json" \
     -d '{
       "token": "abc123...",
       "invoice_id": "INV-001",
       "promised_date": "2024-02-15",
       "amount": 1000,
       "note": "Cash flow issues"
     }'
   ```

2. **Create Dispute**
   ```bash
   curl -X POST https://portal.yourdomain.com/api/portal/disputes \
     -H "Content-Type: application/json" \
     -d '{
       "token": "abc123...",
       "invoice_id": "INV-001",
       "reason_code": "PRICING",
       "detail": "Price discrepancy"
     }'
   ```

```bash
# Enable surcharge policy

  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \

    "pct": 0.03,

  }'
```

## 🚨 Troubleshooting

### Common Issues

**1. Portal Token Invalid/Expired**

- Check session expiration time
- Verify token format (64 hex characters)
- Check for clock skew issues

**2. Payment Gateway Failures**

- Verify gateway credentials
- Check webhook signature validation
- Review DLQ for failed webhooks

**3. Surcharge Calculation Errors**

- Verify policy configuration
- Check rounding precision (2 decimal places)
- Validate min/cap fee logic

**4. Cash Application Not Triggering**

- Verify M24 cash-app service is running
- Check remittance import format
- Review payment matching logic
