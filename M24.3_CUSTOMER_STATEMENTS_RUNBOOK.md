# M24.3 — Customer Statements & Portal Ledger (Order-to-Cash v1.3) Runbook

## Overview

M24.3 is a tight add-on to M24/M24.2 that provides monthly customer statements (PDF/CSV), a customer-facing ledger in the portal, optional finance charges, and fully auditable email dispatch. Zero churn to your existing AR data model; plugs into cash-app, PTP/disputes, and M22 receipts.

## Features

- **Monthly Statement Generation**: Automated PDF/CSV generation with accurate ledger composition
- **Finance Charge Calculation**: Optional APR-based charges with grace periods and minimum fees
- **Email Dispatch**: Tracked statement delivery with retry logic and status monitoring
- **Portal Ledger**: Customer-facing ledger with token-based access and latest statement links
- **Aging Buckets**: Automatic categorization (CURRENT/1-30/31-60/61-90/90+)
- **Audit Trail**: Complete statement run history with artifact storage and checksums

## Architecture

### Database Schema

The system adds 6 new tables to M24:

1. `ar_finance_charge_policy` - Finance charge configuration per company
2. `ar_statement_run` - Statement generation runs with status tracking
3. `ar_statement_line` - Individual ledger lines per customer per run
4. `ar_statement_artifact` - Generated PDF/CSV files with checksums
5. `ar_statement_email` - Email dispatch tracking and status
6. `ar_portal_ledger_token` - Token-based portal access for customers

### Services

- **ArStatementService**: Core statement generation, finance charge calculation, and email dispatch
- **ArPortalLedgerService**: Customer-facing ledger access with token management
- **Ledger Composer**: Builds normalized ledger from invoices, payments, adjustments
- **Finance Charge Calculator**: Computes charges for overdue invoices (simple/daily compounding)
- **Artifact Generator**: Creates PDF/CSV statements with proper formatting

### API Endpoints

**Admin/Ops Routes:**

- `GET/POST /api/ar/finance-charge/policy` — Finance charge policy management (`ar:stmt:policy`)
- `POST /api/ar/statements/run` — Generate statements (`ar:stmt:run`)
- `GET /api/ar/statements/runs` — List statement runs (`ar:stmt:run`)
- `POST /api/ar/statements/email` — Send statement emails (`ar:stmt:email`)
- `GET /api/ar/statements/artifacts/[id]` — Download statement artifacts (`ar:stmt:run`)

**Portal Routes (public token):**

- `POST /api/portal/ledger` — Get customer ledger via token
- `GET /api/portal/statements/latest?token=...` — Get latest statement URLs

## Installation & Setup

### 1. Database Migrations

Run the following migrations in order:

```bash
psql -d $DATABASE_URL -f packages/adapters/db/migrations/0139_ar_finance_charge_policy.sql
psql -d $DATABASE_URL -f packages/adapters/db/migrations/0140_ar_statement_run.sql
psql -d $DATABASE_URL -f packages/adapters/db/migrations/0141_ar_statement_line.sql
psql -d $DATABASE_URL -f packages/adapters/db/migrations/0142_ar_statement_artifact.sql
psql -d $DATABASE_URL -f packages/adapters/db/migrations/0143_ar_statement_email.sql
psql -d $DATABASE_URL -f packages/adapters/db/migrations/0144_ar_portal_ledger_token.sql
psql -d $DATABASE_URL -f packages/adapters/db/migrations/0145_ar_perf_idx4.sql
psql -d $DATABASE_URL -f packages/adapters/db/migrations/0146_ar_view_customer_ledger.sql
```

### 2. Environment Variables

Add to your `.env`:

```bash
# Statement Configuration
STATEMENT_STORAGE_PATH=/tmp/statements
STATEMENT_BASE_URL=https://your-domain.com/api/ar/statements/artifacts

# Email Configuration (reuse existing M15.2 settings)
EMAIL_SERVICE_URL=https://your-email-service.com
EMAIL_API_KEY=your-api-key
```

### 3. RBAC Permissions

Grant the following capabilities to appropriate roles:

```sql
-- For admin/accountant roles
GRANT ar:stmt:policy TO admin_role;
GRANT ar:stmt:run TO admin_role;
GRANT ar:stmt:email TO admin_role;
```

### 4. Seed Data

Create default finance charge policy:

```sql
INSERT INTO ar_finance_charge_policy(company_id, enabled, annual_pct, min_fee, grace_days, comp_method, updated_by)
VALUES ('your-company-id', false, 0.18, 0, 10, 'simple', 'ops')
ON CONFLICT (company_id) DO UPDATE SET annual_pct=EXCLUDED.annual_pct;
```

## Usage Examples

### 1. Configure Finance Charge Policy

```bash
curl -X POST -H "X-API-Key: <id>:<secret>" -H "Content-Type: application/json" \
  -d '{
    "enabled": true,
    "annual_pct": 0.18,
    "min_fee": 5.00,
    "grace_days": 10,
    "comp_method": "simple",
    "present_ccy": "USD"
  }' \
  https://your-domain.com/api/ar/finance-charge/policy
```

### 2. Generate Monthly Statements

```bash
curl -X POST -H "X-API-Key: <id>:<secret>" -H "Content-Type: application/json" \
  -d '{
    "as_of_date": "2024-09-30",
    "present": "USD",
    "finalize": true,
    "include_pdf": true,
    "include_csv": true
  }' \
  https://your-domain.com/api/ar/statements/run
```

### 3. Send Statement Emails

```bash
curl -X POST -H "X-API-Key: <id>:<secret>" -H "Content-Type: application/json" \
  -d '{
    "run_id": "01K6NG30EJRTV4G1N2ZX6YKGFD",
    "resend_failed": true
  }' \
  https://your-domain.com/api/ar/statements/email
```

### 4. Customer Portal Ledger Access

```bash
# Generate token (internal)
curl -X POST -H "X-API-Key: <id>:<secret>" -H "Content-Type: application/json" \
  -d '{
    "company_id": "your-company",
    "customer_id": "customer-123",
    "ttl_minutes": 60
  }' \
  https://your-domain.com/api/ar/portal/ledger/token

# Customer accesses ledger
curl -X POST -H "Content-Type: application/json" \
  -d '{
    "token": "abc123...",
    "since": "2024-07-01",
    "until": "2024-09-30",
    "include_disputes": true
  }' \
  https://your-domain.com/api/portal/ledger

# Get latest statement URLs
curl "https://your-domain.com/api/portal/statements/latest?token=abc123..."
```

## Troubleshooting

### Common Issues

**1. Statement Generation Fails**

- Check database connectivity
- Verify invoice data integrity
- Review finance charge policy configuration
- Check available disk space for artifacts

**2. Email Dispatch Issues**

- Verify email service configuration
- Check customer email addresses
- Review email service quotas
- Monitor bounce rates

**3. Portal Access Problems**

- Validate token expiration
- Check customer permissions
- Verify statement run status
- Review portal base URL configuration

**4. Finance Charge Calculation Errors**

- Verify policy configuration
- Check invoice due dates
- Review grace period settings
- Validate currency conversion

### Performance Optimization

- **Database Indexes**: Ensure proper indexing on statement queries
- **Artifact Storage**: Use efficient storage backend (S3, etc.)
- **Batch Processing**: Process customers in batches for large datasets
- **Caching**: Cache customer master data for frequent access
